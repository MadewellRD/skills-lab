# Handoff Rules

Route follow-up work by type.

- Missing implementation, failing acceptance criteria, or code-level defects: send to `implementation-handoff-desk`.
- Missing tests, inadequate scenarios, or fixture gaps: send to `test-strategy-desk`.
- Requirement ambiguity: send to `product-requirements-desk` or `issue-planning-desk`.
- Architecture ambiguity: send to `architecture-design-desk`.
- Security/privacy blockers: send to `security-threat-desk`.
- Release readiness and rollout work: send to `release-operations-desk`.
- Proof-map or status-doc updates: send to `docs-traceability-desk`.

Every handoff must include source facts, affected requirement IDs, evidence gaps, recommended next action, and halt conditions.
