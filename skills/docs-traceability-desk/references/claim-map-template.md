# Claim Map Template

Use this artifact when the first task is extracting and classifying claims from docs.

```markdown
# Claim Map: <document or corpus>

## Scope

<Document set, repo, branch, PRs, issues, and source boundaries.>

## Claim inventory

| Claim ID | Source | Claim text | Claim type | Evidence needed | Current status |
|---|---|---|---|---|---|

## Claim types

- product behavior
- implementation state
- test coverage
- release status
- architectural decision
- security/compliance control
- operational process
- roadmap/future intent

## Review notes

<Ambiguities, duplicates, contradictions, and follow-up questions.>
```
