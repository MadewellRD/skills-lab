# Proof Map Template

Use this structure for proof maps.

```markdown
# Proof Map: <topic>

## Summary

<What is being proven, what sources were used, and the overall confidence.>

## Source facts

| Source | Type | Date/ref | Relevant section/path | Notes |
|---|---|---|---|---|

## Proof table

| Claim ID | Claim | Status | Supporting evidence | Gaps/conflicts | Action |
|---|---|---|---|---|---|
| CLM-001 | <claim> | supported | <source facts> | none | no action |

## Unsupported or stale claims

| Claim ID | Claim | Why unsupported/stale | Required evidence | Recommended owner/action |
|---|---|---|---|---|

## Downstream handoff

<Notes for implementation-handoff-desk, verification-desk, review-quality-desk, or issue-planning-desk.>
```

Status values: `supported`, `partially supported`, `unsupported`, `stale`, `conflicting`, `unverified`.
