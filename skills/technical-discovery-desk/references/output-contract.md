# Output Contract

When creating an artifact, use a downloadable Markdown file when tools allow it.

## Wrapper

```markdown
# <artifact title>

## How to use this file

Use this artifact as technical discovery input. Keep source facts, assumptions, risks, open questions, and downstream routing intact.

## Artifact

<artifact content>
```

## Artifact types

- `technical-discovery-memo.md`
- `repo-reconnaissance-report.md`
- `feasibility-assessment.md`
- `spike-plan.md`
- `risk-register.md`
- `connector-diagnostic.md`

## Required sections

Every substantive artifact must include:

- Executive summary
- Scope and non-scope
- Source facts
- Assumptions
- Risks and unknowns
- Feasibility or readiness decision
- Downstream handoff

## Evidence block

Include source facts in the artifact or in a companion source-notes section. If citations are unavailable, state what was user-provided and what remains unverified.
