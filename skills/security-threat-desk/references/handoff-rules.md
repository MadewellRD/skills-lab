# Downstream Handoff Rules

## To architecture-design-desk

Send unresolved trust boundaries, data-flow gaps, security requirements, and architectural mitigations.

## To issue-planning-desk

Send confirmed mitigations that need issue decomposition, acceptance gates, labels, priority, and dependencies.

## To review-quality-desk

Send PR-specific review risks, missing tests, changed security-sensitive files, and suggested review comments.

## To verification-desk

Send controls that need evidence, acceptance gates, test requirements, and release-blocking findings.

## To ci-failure-desk

Send security checks or dependency scans that fail in CI, with logs and suspected root cause when available.

## To implementation-handoff-desk

Send bounded implementation-ready mitigation work. Include repo, branch, files, acceptance gates, validation commands, out-of-scope boundaries, and halt conditions.
