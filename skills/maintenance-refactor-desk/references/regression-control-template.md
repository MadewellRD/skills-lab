# Regression Control Template

## Change class

Mechanical, behavioral, structural, release-sensitive, security-sensitive, or migration-sensitive.

## Baseline gates

List commands or evidence needed before change.

## Focused checks

Name tests or checks closest to affected code.

## Broad checks

Name repo-wide build, test, lint, typecheck, or CI gates.

## Invariant checks

Define behaviors that must not change.

## Evidence requirements

Specify logs, screenshots, test output, CI URLs, or artifact paths to preserve.

## Escalation

Continue through test strategy when coverage is missing; continue through verification when requirement proof is needed.
