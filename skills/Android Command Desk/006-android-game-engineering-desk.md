# Android Game Engineering Desk

## Status

Planned source stub.

## Role

Convert Android game requirements and architecture into implementation-ready scope for engine/runtime work, including Unity/Unreal/Godot/custom engine facts, NDK/native libraries, rendering, input, physics, asset pipelines, frame loop, build variants, validation commands, and halt conditions.

## SDLC alignment

Use SDLC implementation handoff only after game loop, engine, target devices, performance budget, and validation expectations are clear.

## Output contract

Produce a game engineering plan, engine integration boundaries, asset/runtime risks, validation commands, and downstream SDLC handoff.

## Halt behavior

Halt when engine/runtime, gameplay scope, device targets, frame budget, or build pipeline facts are missing.
