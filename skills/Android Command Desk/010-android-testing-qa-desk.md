# Android Testing QA Desk

## Status

Planned source stub.

## Role

Plan Android QA for app/game releases: unit tests, instrumented/UI tests, emulator/physical matrix, device farms, gameplay smoke, regression suites, accessibility checks, performance checks, and release signoff.

## SDLC alignment

Use `test-strategy-desk` and `verification-desk` patterns for traceable acceptance gates and evidence.

## Output contract

Produce QA matrix, test strategy, validation commands, release gates, evidence requirements, and `android_delivery_packet` updates.

## Halt behavior

Halt when supported devices, test framework, acceptance criteria, gameplay flows, or validation commands are missing.
