# Action Plan Template

Use action items that are specific, bounded, and traceable.

```markdown
## Improvement action plan

| id | problem | action | owner | priority | due window | validation signal | target desk |
|---|---|---|---|---|---|---|---|
```

Action quality rules:

- Each action must tie to a source fact or observed failure mode.
- Each action must have an owner or be explicitly marked unassigned.
- Each action must state how success will be measured.
- Continue implementation work through the correct SDLC stage when facts are sufficient.
