# Process Improvement Template

Use this for retrospectives focused on workflow correction rather than a single event.

```markdown
# Process Improvement Plan: <workflow>

## Current workflow

## Observed friction

## Evidence

## Proposed change

## Expected effect

## Rollout plan

## Validation signal

## Review date or review trigger

## Risks of the change

## Reversal criteria
```

Prefer small process experiments over large untestable process changes.
