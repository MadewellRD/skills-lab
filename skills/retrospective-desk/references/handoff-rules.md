# Handoff Rules

Route findings by follow-up type:

- Product ambiguity -> `product-requirements-desk`.
- Technical unknown or feasibility miss -> `technical-discovery-desk`.
- Design flaw -> `architecture-design-desk`.
- Poor issue breakdown -> `issue-planning-desk`.
- Implementation or PR follow-up -> `implementation-handoff-desk`.
- Review quality issue -> `review-quality-desk`.
- Missing coverage -> `test-strategy-desk`.
- Acceptance or evidence gap -> `verification-desk`.
- CI instability -> `ci-failure-desk`.
- Release process issue -> `release-operations-desk`.
- Deployment or rollout issue -> `deployment-desk`.
- Observability gap -> `observability-readiness-desk`.
- Production incident follow-up -> `incident-response-desk`.
- Docs drift -> `docs-traceability-desk`.
- Refactor or dependency debt -> `maintenance-refactor-desk`.

Each handoff note must include the source fact, the recommended desk, and the expected artifact.
