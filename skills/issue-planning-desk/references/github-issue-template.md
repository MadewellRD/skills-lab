# GitHub Issue Draft Template

Use this for issue bodies intended for GitHub.

```markdown
## Summary

<One paragraph describing the required change.>

## Source context

- Requirement:
- Design or discovery source:
- Related files or modules:
- Related issues or PRs:

## Scope

- Implement:
- Update:
- Verify:

## Acceptance criteria

- [ ] Criterion 1
- [ ] Criterion 2
- [ ] Validation evidence is attached or documented

## Dependencies

- Depends on:
- Blocks:

## Out of scope

- Deferred work:

## Validation

- Command:
- Manual evidence:

## Handoff notes

- Suggested owner or agent:
- Suggested follow-up PR prompt type:
```

Issue titles should be imperative and scoped, for example `add requirement trace IDs to exported PRD` rather than `requirements stuff`.
