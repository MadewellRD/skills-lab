# iOS Architecture Design Desk

## Status

Planned source stub.

## Role

Define iOS architecture for app and game work: modules, layers, state, data flow, storage, networking, engine boundaries, native/plugin boundaries, background tasks, API contracts, migration impact, and ADR decisions.

## SDLC alignment

Use `architecture-design-desk` patterns for ADRs, interface contracts, migration planning, and architectural risks.

## Output contract

Produce architecture decisions, interface boundaries, migration notes, diagrams-as-text where useful, risks, validation expectations, and `ios_delivery_packet` updates.

## Halt behavior

Halt when repo architecture, target framework, runtime constraints, service contracts, or migration scope are unknown.
