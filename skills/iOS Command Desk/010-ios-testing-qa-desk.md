# iOS Testing QA Desk

## Status

Planned source stub.

## Role

Plan iOS QA for app/game releases: unit tests, UI tests, simulator/physical matrix, TestFlight smoke, gameplay smoke, regression suites, accessibility checks, performance checks, and release signoff.

## SDLC alignment

Use `test-strategy-desk` and `verification-desk` patterns for traceable acceptance gates and evidence.

## Output contract

Produce QA matrix, test strategy, validation commands, release gates, evidence requirements, and `ios_delivery_packet` updates.

## Halt behavior

Halt when supported devices, test framework, acceptance criteria, gameplay flows, or validation commands are missing.
