# v0.1.0 — Initial SDLC Command Desk Suite

Status: draft release notes

## Summary

This release establishes the first SDLC Command Desk skill suite for token-efficient, connector-grounded AI-native software delivery.

The suite is designed for vibe coders, solo builders, and AI-native engineering teams that want to move quickly while preserving software lifecycle discipline.

## Core principles

- Route ambiguous work through `sdlc-command-desk`.
- Use lifecycle-specific desks for stage work.
- Use `implementation-handoff-desk` only after upstream ambiguity has been reduced.
- Treat GitHub as source of truth for repository state.
- Use docs and communication connectors for product, decision, and operational context.
- Halt or produce diagnostics instead of inventing missing facts.
- Produce compact, reusable Markdown artifacts.
- Help coding agents spend tokens on implementation, tests, and validation instead of process rediscovery.
