# v0.1.1 - Workflow-linked SDLC Command Desk Suite

Status: packaged release candidate

## Summary

This release updates the SDLC Command Desk suite so the skills operate as one continuous lifecycle workflow instead of isolated one-off prompts.

The main behavioral change is that `sdlc-command-desk` now acts as an end-to-end orchestrator. It should classify the user's request, run the necessary lifecycle stages, preserve a workflow packet, and continue until the target outcome is reached or a real halt condition is encountered.

## What changed

- `sdlc-command-desk` is now a workflow orchestrator, not just a router.
- Every skill now includes `references/suite-workflow-contract.md`.
- Every stage must preserve or emit a workflow packet.
- Skills should continue to the next stage when source facts are sufficient.
- Skills should not stop with only a bare next-desk recommendation.
- Workflow stops must use `Workflow Halt` with exact missing facts and a resume prompt.
- `sdlc-command-desk` now includes `references/stage-contracts.md`.
- `sdlc-command-desk` now includes `scripts/run_sdlc_workflow.py` for deterministic workflow sequencing.
- Historical `pr-command-desk` references were normalized to `implementation-handoff-desk`.

## Workflow packet

Each workflow-aware output should preserve this shape:

```yaml
workflow_packet:
  workflow_id: "user-or-generated-id"
  mode: "single_stage | workflow_run | resume | halt"
  current_stage: "stage-name"
  completed_stages:
    - "stage-name"
  next_stage: "stage-name-or-none"
  target_stage: "stage-name-or-none"
  source_facts:
    - fact: "source-backed fact"
      source: "github | docs | user | connector | unknown"
  decisions:
    - "decision made at this stage"
  open_questions:
    - "question blocking later work"
  artifacts:
    - "artifact name or path"
  halt_conditions:
    - "condition that requires stopping"
  ready_to_continue: true
```
